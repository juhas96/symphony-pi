Smoke behavior:
- pi extension smoke starts pi RPC with this package via `--extension`, verifies `/symphony:*` commands, invokes `/symphony:validate`, skips if pi is unavailable.
- Codex schema smoke runs `codex app-server generate-json-schema --out <tmp>`, checks thread/turn start schema fields, skips if Codex unavailable.
- real Codex smoke creates a temp workspace and sends "Reply with OK and do not modify files." through `CodexAppServerClient`, requires session_started and turn_completed, skips if Codex/auth/model unavailable.
- Beads E2E smoke initializes temp Beads project, creates safe issue, writes Beads WORKFLOW.md, runs CLI `--once` with fake Codex app-server, verifies workspace creation.
- Linear live smoke is read-only/opt-in with `PI_SYMPHONY_LIVE_LINEAR=1`, `LINEAR_API_KEY`, `LINEAR_PROJECT_SLUG`; checks candidate fetch, empty state behavior, terminal fetch, and state refresh by GraphQL ID.
- Jira live smoke is read-only/opt-in with `PI_SYMPHONY_LIVE_JIRA=1`, `JIRA_EMAIL`, `JIRA_API_TOKEN`, `JIRA_ENDPOINT`, and `JIRA_PROJECT_KEY` or `JIRA_JQL`; uses page_size=1 to exercise pagination and refreshes by key.

- configured tracker secrets and obvious token/password patterns are redacted before artifact writes.
- `.symphony/runs/` is mutable local output and should stay ignored by git.
- artifact paths are included in running/retry issue snapshots and HTTP API responses.
