Final implementation additions after previous checkpoint:

1. `pi-symphony-kbh.7` — Contextual action menu expansion and confirmations
- Action menu is now tab-contextual.
- Queue actions include run selected/first eligible and issue URL.
- Logs actions include open current log path, cycle severity filter, jump latest error, clear search.
- Runs/Issue actions include refresh tracker issue details, open artifact path, workspace path, run log path, raw result toggle.
- Config actions include raw config toggle and dashboard open.
- Disabled actions show reasons.
- Stop daemon confirms when active workers are running.
- Tests cover action labels and stop confirmation flow.

2. `pi-symphony-kbh.6` — Responsive layout and visual hierarchy polish
- Queue rows use compact/minimal layouts at narrower widths.
- Running rows hide PID/turn/tokens in compact mode.
- Runs rows hide terminal reason column in compact mode.
- `<40` columns now shows a narrow-terminal Overview/Help warning instead of trying to render the full app.
- Representative render-width assertions were added.

3. `pi-symphony-kbh.5` — Richer Issue detail with manual tracker refresh
- Added `SymphonyOrchestrator.refreshIssueDetails(issue)`.
- Added `SymphonyConsoleDataProvider.refreshIssueDetails(issue)`.
- Issue view now distinguishes `Source: snapshot/artifact` from `Source: tracker refresh at <timestamp>`.
- Manual refresh is available through the Issue action menu.
- Refreshed details are cached in memory for the current console session.
- Failed refresh stores/display refresh error.
- Tests cover action, refresh, cache/source behavior.

4. `pi-symphony-kbh.9` — Decide lightweight UI state persistence
- Decision doc: `docs/decisions/2026-05-03-symphony-console-ui-state-persistence.md`.
- Decision: do not persist UI state in v1.1; keep session-only/in-memory.
- Rationale: avoids writing issue identifiers, paths, filters, logs, or artifact traces to repo/session files.

5. `pi-symphony-kbh.10` — Mouse support feasibility spike
- Spike doc: `docs/spikes/symphony-console-mouse-support.md`.
- Finding: pi TUI public Component API exposes keyboard/string `handleInput`, but no documented typed mouse/click coordinate API or overlay routing contract.
- Decision: defer mouse support; no v1.1 production behavior depends on mouse.
